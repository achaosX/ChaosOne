# ChaosOne Roadmap

## MVP 0.1

- Local web panel
- Windows start script
- Basic install script with shortcuts
- Personality and provider configuration
- OpenAI chat path
- Provider Hub with local/cloud privacy labels
- Chaos Local path through Ollama-compatible local runtime
- OpenClaw identity import

## MVP 0.2

- First-run setup wizard polish
- Local-first onboarding with explicit cloud opt-in
- Chaos Local runtime packaging plan without requiring Ollama as final UX
- API key storage via Windows DPAPI / Credential Manager
- Memory editor
- Import existing OpenClaw workspace identity files into runtime prompts
- Provider registry refresh workflow

## MVP 0.3

- Background tray process
- Real installer package
- Auto-update strategy
- OpenClaw bridge layer for tools, sessions, skills and nodes
- Cloud connectors for Anthropic, Gemini, Mistral and Azure OpenAI

## Product Direction

ChaosOne should feel like a simple personal assistant product while keeping the deeper OpenClaw mechanics available under the hood.

Privacy direction: ChaosOne should be local-first. Cloud providers are powerful and useful, but should always be explicit, visible and opt-in.
